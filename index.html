Java集合

发表于 2020-08-28 更新于 2020-08-29

  • List、Set、Map都是接口，前两者继承Collection接口，Map为独立接口

  • Set下有HashSet、LinkedHsahSet、TreeSet

  • List下有ArrayList、Vector、LinkedList

  • Map下有Hashtable、LinkedHashMap、HashMap、TreeMap

  • Collection接口下还有个Queue接口，有PriorityQueue类

注意：

  • Queue接口与List、Set同一级别，都是继承了Collection接口，看图就可以知道LinkedList可以实现List接口也可以实现Queue接口。Queue接口窄化了对LinkedList的方法的访问权限（在方法中的参数类型如果是Queue时，就完全只能访问Queue接口所定义的方法了，而不能直接访问LinkedList的非Queue的方法），以使得只有恰当的方法才可以使用。
  • SortedSet是一个接口，它里边中的元素一定是有序的。

总结

Connection接口：

List 有序，可重复

  • ArrayList

    优点： 底层数据结构是数组，查询快，增删慢

    缺点： 线程不安全，效率高

  • Vector

    优点： 底层数据结构是数组，查询快，增删慢

    缺点：线程安全，效率低

  • LinkedList

    优点：底层数据结构是链表，查询慢，增删快

    缺点：线程不安全，效率高

Set 无序、唯一

  • HashSet

    底层数据结构是哈希表。（无序、唯一）

    通过hashCode（）和equals（）来保证元素的唯一性

  • LinkedHashSet

    底层数据结构是链表和哈希表。

    链表保证了元素的有序、哈希表保证了元素的唯一

  • TreeSet

    底层数据结构式红黑树（唯一、有序）

    1. 如何保证元素的排序

      自然排序、比较器排序

    2. 如何保证元素的唯一性

      根据比较的返回值是否是0来决定

Collection集合使用的选择：

元素如果唯一选择Set，如果需要排序选择TreeSet或者LinkedHashSet，否则选择HashSet。

元素不唯一选择List，如果需要线程安全选择Vector，否则选择ArrayList、LinkdedList，查询多选择ArrayList，增删多选择LinkedList

Map有三个比较重要的实现类，分别是HashMap、TreeMap和HashTable

  • TreeMap是有序的，HashMap和HashTable是无序的。

  • Hashtable的方法是同步的，HashMap的方法不同步。

  • HashMap的线程不安全，Hashtable的线程安全。

  • HashMap的效率高，HashTable的效率低。

  • Hashtable不允许null值，HashMap允许null值（key和value都允许）

  • 父类不同：Hashtable的父类是Dictionary、HashMap的父类是AbstractMap

重点问题重点分析

一、

TreeSet、LinkedHashSet、HashSet的区别

  • TreeSet、LinkedHsahSet、HashSet在Java中都是实现Set数据结构
  • TreeSet的主要功能用于排序
  • LinkedHashSet的主要功能是用于保证FIFO即有序的集合
  • HashSet只是通用的存储数据的集合

相同点

  • 重复元素: 因为三者都实现Set interface，所以三者元素都是唯一的
  • 线程安全：三者都不是线程安全，如果要使用线程安全可以Collection.synchronizedSet()

不同点

  • 性能和速度：HashSet插入数据最快，其次LinkedHashSet、最慢的是TreeSet（TreeSet实现了内部排序）

  • 排序：HashSet不保证有序，LinkedHashSet保证FIFO按插入顺序排序，TreeSet内部实现排序，可以自定义排序规则

  • null：HashSet和LinkedSet允许存在null数据，但是TreeSet中插入null数据会报空指针。

Hexo博客搭建

准备环境：安装git、hexo框架

一、下载git 官方地址为：https://git-scm.com/download/win

根据自己的系统选择git的版本，跟随指引安装

安装完成后 win+r cmd 输入git --version

git 安装完成。

二、注册git帐号 网址是http://github.com

打开gitbash，输入ssh-keygen -t rsa -C “你注册时用的邮箱”

在上边路径找到文件id.rsa.pub，复制内容。

打开github.com

点击settings，选择ssh and GPG keys，在ssh keys点击New SSH key，将id.rsa.pub中的内容复制到上边。

在gitbash中输入 ssh -T git@gitgithub.com

这样我们就准备好了git的布置。

三、安装Node.js

hexo是基于Node的一种框架，所以我们需要安装Node 官方网址https://nodejs.org，下载好安装包直接安装即可。

按账号在命令行输入 node -v 来检查我们安装好的node的版本。

安装国内的镜像 npm install -g cnpm --registry=https://registry.npm.tooboo.org (科学上网可以无视)

安装完成可以通过``cnpm -v`查看版本

四、安装hexo框架

cnpm install -g hexo-cli

通过 hexo -v来查看版本号

在你博客安装位置，创建文件夹blog，命令行进入blog，输入hexo init

安装完成后输入hexo s ，通过localhost:4000端口来对博客进行本地预览。

五、博客的书写

建议使用Typora等文件进行书写，Typora地址 https://www.typora.io/

通过hexo n "文章名字" 来创建新的文章，建议通过hexo代码来创建文章，在通过Typora来书写博客， hexo g 来发表自己的文章。

对Typora图片管理的设置可以使得博客上传之后图片不会出现问题。

六、hexo博客的部署

在github上部署自己的个人博客，创建新的仓库

仓库名字与你github帐号名字一致。

安装hexo的git插件cnpm install --save hexo-deployer-git

修改blog目录下_config.yml文件

打开_config.yml 拉到最底下

修改成这个样子repo 写上你的github网址

同时hexo博客语言的修改也是在这个配置文件中，language： 修改成你下载的主题适配的语言。

做好以上配置 我们在命令行输入 hexo d 将本地博客上传到github，我们就可以通过github网址来访问了。

七、修改主题

通过git clone 命令来克隆大佬的主题

例如：git clone https://github.com/smackgg/hexo-theme-smackdown themes/smackdown

从github克隆下来的文件在blog目录下的thems文件夹中。

我们通过修改_config.yml文件来修改主题。
